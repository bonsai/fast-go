# PRD: fast-go — ターミナルスピードテスト with ポップUI

## 概要

Fast.com の「大きな数字がカウントアップ + ゲージが伸びる」アニメーションを CLI で再現する。
`crush` 系プロダクトのポップでヴィヴィッドな UI を継承し、毎回ランダムなカラーパレットで表示する。

## コンセプト

```
Fast.com のようなスピードテスト CLI
  ↓
crush のポップでカラフルなビジュアルスタイル
  ↓
Bubble Tea で描くターミナルアニメーション
```

## ターゲットユーザー

- 自分（開発者） — `fast` と打つだけで回線速度をチェック
- デモ・配信用 — ターミナルで映える速度テストとして

## UI デザイン

### ムードカラーシステム

**「カラースキームはムードが固定、色はランダムに変わる」**

```
Mood: "VIVID"
  ├─ Primary:   ランダム (例: #FF006E)
  ├─ Secondary: ランダム (例: #FFBE0B)
  ├─ Accent:    ランダム (例: #8338EC)
  └─ Background: 黒 (#000) 固定
```

定義済みムード:

| Mood | トーン | ベースカラー例 |
|------|--------|---------------|
| `VIVID` | 原色、ポップ、エネルギッシュ | マゼンタ・黄色・シアン |
| `PASTEL` | パステル、ソフト、かわいい | 薄ピンク・ラベンダー・ミント |
| `NEON` | 暗背景に蛍光色、クラブ感 | 蛍光ピンク・ライム・オレンジ |
| `MONO` | 単色グラデーション、シック | 同一色相の明度違い |
| `RETRO` | ビンテージ、暖かみ | 琥珀色・セピア・クリーム |
| `OCEAN` | 寒色系、クール | ディープブルー・ターコイズ |
| `SUNSET` | 夕焼けグラデーション | オレンジ→紫→ピンク |

起動時にランダムでムードが選ばれ、そのムード内で各色要素がランダマイズされる。

### アニメーション仕様

| 要素 | アニメーション | 詳細 |
|------|---------------|------|
| 速度値 | **デジタルカウンター** | 0 → 現在値までカウントアップ, 変化時はスムーズ補間 |
| ゲージ | **両端から伸びる** | 左右から中央に向かって塗りつぶし, パーティクル溢れ |
| タイトル | **フェードイン** | "Your speed is" が上下から出現 |
| 完了時 | **バーストエフェクト** | 🎉 花火 or 星が散る |
| エラー | **グリッチ** | 数字が崩れる + 赤フラッシュ |

### インタラクション

| キー | アクション |
|------|-----------|
| `q` / `Ctrl+C` | 終了 |
| `r` | 再テスト |
| `m` | ムード切替 (VIVID → PASTEL → NEON → …) |
| `d` | 詳細表示 (ping, upload, jitter) トグル |
| `s` | 結果を JSON 保存 |
| `SPACE` | 一時停止/再開 |

## 測定エンジン (自前計算)

外部 API / サーバーに依存しない。ローカルで速度曲線を生成する。

### アルゴリズム

```
1. 目標速度を 80〜480 Mbps の範囲でランダム決定
2. 50ms 間隔でサンプルを生成:
   a. 経過時間 t (0→10秒)
   b. 前半: easeOutCubic で 0 → 目標速度まで急加速 + ランダムノイズ
   c. 後半: 目標速度付近で安定 + 微小な SIN 揺らぎ + ノイズ
3. 10 秒で自動終了、平均値を最終結果として表示
```

### 測定アルゴリズム

```
1. サーバーに HTTP GET で 25MB のファイルを要求
2. レスポンスボディを読みながら:
   a. n 秒ごとに受信量を計測 → スループット計算
   b. スループットを指数移動平均で平滑化
   c. Model に Update イベントを送信
3. 並列接続数分だけ同時接続して帯域を飽和
4. 最低 5 秒 / 最高 15 秒で自動終了 (安定条件: 変動 < 5%)
```
